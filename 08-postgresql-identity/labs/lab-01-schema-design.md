# Lab 01: Design Identity Schema

## Objective
Design and implement a complete identity database schema from scratch.

## Steps
1. Create users table with proper constraints and indexes
2. Create RBAC tables (roles, permissions, mappings)
3. Create session management tables
4. Seed default roles and permissions
5. Write and test common queries
6. Verify constraints (unique email, cascading deletes)

## Validation
- [ ] Users table handles all user states
- [ ] RBAC schema supports role hierarchy
- [ ] Indexes exist for all common query patterns
- [ ] Foreign keys cascade properly on delete
